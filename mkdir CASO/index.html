01 · Fundamentos

¿Qué es Security Misconfiguration?

Ocurre cuando la configuración de seguridad no se define, implementa o mantiene correctamente, dejando sistemas expuestos a ataques.

Credenciales por Defecto

Contraseñas y usuarios que vienen preconfigurados en sistemas y no son cambiados: admin/admin, root/toor.

Archivos Sensibles Expuestos

Archivos de configuración accesibles públicamente: .env, config.yml, backup.sql.

Mensajes de Error Verbosos

Stack traces y errores detallados que revelan rutas, versiones de software e información interna del sistema.

Listado de Directorios

Servidores web que muestran el contenido de directorios cuando no hay archivo index, exponiendo toda la estructura.

Headers de Seguridad Faltantes

Ausencia de cabeceras HTTP críticas como CSP, HSTS, X-Frame-Options que protegen contra ataques del lado cliente.

Modo Debug en Producción

Aplicaciones con modo debug activo en producción exponen código fuente, variables de entorno y paneles de administración.

        03 · Defensa
        
Estrategias de Mitigación

        
Cómo proteger tu aplicación contra cada tipo de Security Misconfiguration.

      


      

        

          

            ❌
            
Configuración Vulnerable

          

          

            
Python / Django

            
# settings.py - ¡INSEGURO!
DEBUG = True              # Expone info sensible
SECRET_KEY = "django-insecure-abc"
ALLOWED_HOSTS = ['*']  # Acepta cualquier host

# Contraseña de BD hardcodeada
DATABASES = {
  'default': {
    'PASSWORD': 'admin123',
  }
}

# Sin HTTPS forzado
SECURE_SSL_REDIRECT = False
SESSION_COOKIE_SECURE = False

          

          

            
Nginx

            
# nginx.conf - ¡INSEGURO!
server {
  listen 80;           # Sin HTTPS
  autoindex on;        # Lista directorios
  
  # Sin headers de seguridad
  server_tokens on;    # Revela versión
}

          

        


        

          

            ✅
            
Configuración Segura

          

          

            
Python / Django

            
# settings.py - SEGURO
DEBUG = False             # Desactivado en producción
SECRET_KEY = os.environ['SECRET_KEY']
ALLOWED_HOSTS = ['midominio.com']

# Credenciales desde variables de entorno
DATABASES = {
  'default': {
    'PASSWORD': os.environ['DB_PASSWORD'],
  }
}

# HTTPS y cookies seguras
SECURE_SSL_REDIRECT = True
SESSION_COOKIE_SECURE = True
SECURE_HSTS_SECONDS = 31536000

          

          

            
Nginx

            
# nginx.conf - SEGURO
server {
  listen 443 ssl http2;
  autoindex off;
  server_tokens off;   # Oculta versión
  
  # Headers de seguridad
  add_header X-Frame-Options DENY;
  add_header X-Content-Type-Options nosniff;
  add_header Content-Security-Policy "default-src 'self'";
  add_header Strict-Transport-Security "max-age=31536000";
}
